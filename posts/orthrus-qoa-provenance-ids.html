ORTHRUS — Provenance IDS의 귀속 품질(QoA) 문제

2026-07-11 논문리뷰보안GNN

탐지 정확도가 아니라 분석가 검수 부담(QoA)을 1급 목표로 재정의한 PIDS — stateless 시간 GNN으로 검수 노드를 5자릿수 줄이고, 엄밀 노드 라벨로 재평가하면 기존 5개 시스템의 precision이 0에 수렴함을 보인 논문.

침입 탐지 시스템이 하루에 경보 수십만 개를 뱉으면, 분석가는 무엇을 하게 될까. 대부분은 아무것도 못 한다. 진짜 공격 노드 몇 개가 무관한 노드 수천~수십만 개에 파묻혀, 사람이 다 열어볼 수가 없기 때문이다. 이 "경보 피로(alert fatigue)"는 탐지율을 아무리 올려도 사라지지 않는 병목이다. ORTHRUS는 바로 이 지점을 문제로 삼는다 — 얼마나 잘 잡느냐가 아니라, 분석가가 실제로 몇 개를 들여다봐야 하느냐를 1급 목표로 끌어올린 논문이다.

저자: Baoxiang Jiang, Tristan Bilot, Nour El Madhoun, Khaldoun Al Agha, Anis Zouaoui, Shahrear Iqbal, Xueyuan Han, Thomas Pasquier · 원문: ORTHRUS: Achieving High Quality of Attribution in Provenance-based Intrusion Detection Systems (USENIX Security 2025)

TL;DR

  • 프로비넌스 IDS(PIDS)는 탐지는 잘하지만 귀속 품질(QoA)이 낮다 — 공격과 무관한 노드까지 수천~수십만 개를 경보해 분석가를 소진(alert fatigue)시킨다. ORTHRUS는 QoA를 1급 설계·평가 목표로 삼는다.
  • 핵심 기여:
    • 경량 stateless 시간 GNN 인코더 + 엣지타입 재구성 디코더로 노드 단위 이상탐지(상태 메모리 없이 최근 N개 엣지만 샘플링 → Kairos 대비 인코더 메모리 ~1/3).
    • 자동 임계값 + 클러스터링 + 인과분석 재구성으로 검수 대상을 수십 개 "attack summary graph"로 압축.
    • 검수 노드를 5 자릿수(orders) 감소: E5-CADETS에서 ORTHRUS가 검수할 노드 12개 vs ThreaTrace ~3,104,109개(≈5 orders↓).
    • 엄밀한 노드 단위 GT(41~123개)를 새로 라벨링하고 precision·MCC로 재평가 — 이 기준에서 5개 baseline이 거의 전부 precision≈0.
주의 — 새 지표는 QoA(정성)+precision+MCC이며 "ADP"는 원문 지표명이 아니다(§5.1 이진 탐지를 가리키는 것으로 추정).

1문제 정의

먼저 이 시스템들이 무엇 위에서 도는지 짚자. PIDS는 OS가 남긴 실행 이력을 그래프로 보고 그 위에서 공격을 찾는다.

  • 대상: OS 실행 이력을 담은 프로비넌스 그래프(노드=프로세스/파일/netflow, 엣지=시스템콜 상호작용+타입+타임스탬프) 위의 침입 탐지·귀속.

문제는 이 그래프 위에서 "잘 탐지하는 것"과 "잘 귀속하는 것"이 다르다는 데 있다.

  • 기존 PIDS의 공백 — 탐지≠귀속:
    • 이상탐지 PIDS는 정확도는 높지만 공격 귀속 품질(QoA)이 낮아 무관한 노드까지 대량 경보한다.
    • 근원 ①: 과거 평가전략(neighborhood 2-hop / batch / source)이 악성 노드를 수천~수십만 개로 과대추정한다.
    • 근원 ②: 극심한 불균형(악성:정상 = 1:1만 ~ 1:100만)에서 accuracy·F1·AUC가 오도된다.
  • 목표: 분석가가 실제로 조사해야 할 노드 수를 최소화하면서 모든 공격을 잡는, 귀속 품질 중심의 탐지.

2관련 연구와 차별점

ORTHRUS는 이 분야를 새로 연 게 아니라, 기존 방법 갈래를 QoA 렌즈로 다시 배치한다. 어디에 서 있는지부터 보자.

  • 방법 갈래 (출처: related-work 웹 수집 노트):
    • TGN/state-memory 계열 — Kairos(2024 S&P). ORTHRUS는 이를 stateless로 경량화.
    • 정적 GNN 계열 — ThreaTrace(2022 TIFS, GraphSAGE), MAGIC(2024 USENIX, GAT masked AE), Flash(2024 S&P, Word2vec+GNN).
    • 시퀀스 AE 계열 — SIGL(2021 USENIX, Graph-LSTM).
  • 차별점 3:
    • 노드 단위 탐지 + 인과분석 공격 재구성(attack summary graph)을 결합한 최초 PIDS라고 주장.
    • QoA/분석가 부담을 1급 설계·평가 목표로 다루는 사실상 유일한 연구(Kairos·Flash는 요약그래프로 부분 대응).
    • baseline 자체 보고 수치와 ORTHRUS 수치는 서로 다른 GT/평가전략 산물이라 직접 비교 불가.

이 마지막 차별점이 왜 중요한지는 아래 표가 보여준다. 같은 시스템이라도 "자기 평가전략"과 "ORTHRUS의 엄밀 평가"에서 전혀 다른 얼굴이 나온다.

3핵심 아이디어 / 방법

그럼 ORTHRUS는 어떻게 검수 노드를 수천에서 수십으로 줄일까. 아이디어는 세 갈래로 나뉜다.

  • 직관 3갈래: ① 상태 메모리 없이 최근 N개 엣지만 stateless 샘플링하는 경량 GNN 인코더, ② 정상 데이터로 엣지 타입을 재구성하도록 자가지도 학습 → 추론 시 재구성 오차가 이상 점수, ③ 오차 큰 노드만 자동 임계+클러스터링으로 소수만 남기고 인과 의존성 분석으로 공격 경로 재구성. 노드 수를 수천 → 수십(GT 41~123)으로 줄여 QoA↑.

이 세 직관은 아래 5단계 파이프라인으로 구현된다. 핵심 기여는 ③(경량 stateless 시간 GNN)에 몰려 있다.

파이프라인 5단계

  • ① 그래프 구성: 원시 로그(ETW/Linux Audit/CamFlow/eAudit) 파싱 → 3종 엔티티 + 이벤트 방향그래프. Causality Preserved Reduction으로 중복 엣지 프루닝(순서 보존; 세부는 인용 위임).
  • ② 엣지 특징화: 엣지 (u,v)에 5개 특징 concat → e_uv. 타입 one-hot(이벤트 t_uv, 엔티티 t_u·t_v) + 노드 속성(파일경로·명령·IP를 토큰화 후 Skip-gram Word2vec 임베딩 평균).
$$\mathbf{x}_v = \sum_{i=1}^{n} \mathbf{w}_i \tag{1}$$
주의 — 본문은 "평균"이라 하나 식(1)은 합 — 1/n 정규화 누락 추정, 원문 미해소.
  • ③ 시간 그래프 학습:
    • Stateless temporal sampling — v로 들어온 과거 엣지 \(S_t(v)=\{(u,v)\in E \mid t_{uv}<t\}\) (식2) 중 최근 N개만 \(S_N(v)=\mathrm{SAMPLE}(S_t(v),N,T,t)\) (식3), N∈[10,20].
      • Kairos와 핵심 차별점 — 엔티티별 메모리 벡터를 저장하지 않아 메모리 ~1/3.
    • GNN 인코더(GraphTransformer 변형, Shi et al., z=32):
$$\alpha_{u,v} = \mathrm{softmax}\!\left(\frac{(\mathbf{W}_3\mathbf{x}_v)^\top(\mathbf{W}_4\mathbf{x}_u + \mathbf{W}_5\mathbf{e}_{uv})}{\sqrt{d}}\right) \tag{5}$$ $$\mathbf{h}_v = \mathbf{W}_1\mathbf{x}_v + \sum_{(u,v)\in S_N} \alpha_{u,v}(\mathbf{W}_2\mathbf{x}_u + \mathbf{W}_5\mathbf{e}_{uv}) \tag{6}$$

디코더(엣지 타입 재구성) — 정상 데이터 전 엣지에 L_uv 최소화, 추론 시 높은 L_uv가 이상 점수:

$$\hat{\mathbf{y}}_{uv} = \sigma(\mathbf{W}_g[\mathbf{W}_s\mathbf{h}_u, \mathbf{W}_d\mathbf{h}_v]) \tag{7}$$ $$L_{uv} = \mathrm{CE}(\hat{\mathbf{y}}_{uv}, \mathbf{y}_{uv}) \tag{8}$$
  • ④ 이상탐지(2단계, 자동화):
    • Automatic Thresholding — 학습에 안 쓴 하루치 정상 validation의 최고 이상점수를 임계값으로 자동 설정, 초과 노드 = 의심.
    • Outlier Clustering — 의심 노드에 K-means(K=2) → 고득점(악성)/저득점(benign anomaly) 분리, 후자 제외 → FP 감소.
  • ⑤ 공격 재구성: 플래그 노드 p마다 15분 시간창 서브그래프 → DAG 변환 → backward(entry)/forward(exit) 추적. 세 점수(min-max 정규화):
$$f_D(u) = \frac{\mathrm{OutDegree}(u)}{\mathrm{InDegree}(u)} \tag{9}$$ $$f_A(u) = \frac{1}{|E_u|}\sum_{uv\in E_u} L_{uv} \tag{10}$$ $$f_C(e) = \frac{1}{|V_e|}\sum_{u\in V_e}(\hat{f}_D(u) + \hat{f}_A(u)) \tag{11}$$

최고 f_C의 critical entry/exit 선택 → 두 dependency graph의 합집합을 attack summary graph로 보고. (L_uv는 §3 디코더 식8에서 온 값을 재사용)

평가 지표 (방법 관점)

  • QoA (Quality of Attribution) — 개념적 정의: IDS 출력 조사·근본원인 파악·오경보 기각에 드는 분석가 노력. 형식 공식 없음. 대리 측정 = 검수 노드 수(작을수록↑) + precision.
  • MCC(식12) — 불균형에 강건한 주 비교 지표. 데이터셋 의존적이라 동일 데이터셋 내 비교만 가능. accuracy/AUC/F1은 불균형에서 과낙관적이라 배제.
  • 평가전략 형식화(Table 1) — Neighborhood/Batch/Source 셋 다 수천 노드 과대추정. ORTHRUS 수작업 GT는 41~123 노드.
주의 — "ADP"는 본문에 없음(grep 확인). 실제 지표는 QoA(개념)·precision·MCC(식12) + §5.1 이진 "detected" 여부.

4실험

이제 실제로 숫자가 그 약속을 지키는지 본다. 셋업부터.

셋업

  • 데이터셋: DARPA TC Engagement 3 & 5, 6개 서브셋(CADETS/THEIA/CLEARSCOPE). 극심한 불균형(악성 ~1:1만 ~ 1:100만). 하루치 benign을 validation으로 예약(자동 임계값용).
  • 모델: GraphTransformer 변형 attention GNN + 엣지타입 예측 디코더(benign만 자가지도). N=10~20, z=32, Word2vec x=32, lr=1e-4, node dropout 25%, 재구성 시간창 15분.
  • 베이스라인 5종: Kairos(S&P'24), ThreaTrace(TIFS'22), SIGL(USENIX'21, 재구현), MAGIC(USENIX'24, GT 라벨 의존 임계), Flash(S&P'24). 변형: ORTHRUS-full(전체) / ORTHRUS-ano(재구성 §4.5 제외).
  • 메트릭: precision(모든 공격 탐지 전제 최우선) + MCC(불균형 강건, 동일 데이터셋 내). F1/AUC/accuracy 배제.

공격 경로 재구성 (full vs ano)

재구성(⑤)을 켜고 끄는 건 trade-off다. TP를 얻는 대신 약간의 FP를 감수한다.

  • 재구성을 켜면 TP가 크게↑, 약간의 FP를 감수: E3-CADETS ano 10TP/0FP(MCC0.38) → full 25TP/23FP(MCC0.44); E3-THEIA ano 8TP/0FP → full 48TP/11FP(MCC0.57). 재구성은 선택적(analyst 필요 시).

5한계 · 비판적 검토

수치는 인상적이지만, 그 수치가 어떻게 만들어졌는지를 뜯어보면 이야기가 복잡해진다. 먼저 맥락 하나를 짚어야 한다.

맥락 주의: 지목된 "USENIX ;login: 2025 재현성 비판"은 검색상 ORTHRUS 저자 본인(Bilot, Pasquier) 저작으로 확인 — 독립 제3자 감사가 아니다.

저자 명시 한계

  • FP가 악성 노드 1-hop 이웃에 몰림(§6) — GNN 메시지 패싱이 악성 신호를 이웃에 전파. E5-CLEARSCOPE ORTHRUS-ano FP 7개 중 5개가 공격 노드와 같은 IP netflow. "better FP mitigation remains open."
  • 캡처 메커니즘 의존/E5-CADETS 저성능(§6) — 캡처 시스템별 정보 누락(CADETS 경로 대부분 누락), DARPA 크래시로 인한 요소 누락 인정.
  • 개념 드리프트/재훈련 미평가(§6) — 필요성은 인정하나 데이터셋 부재로 future work.
  • GT 보수적 누락(Appendix C) — 불확실 요소 배제 → 탐지 성능 과소평가 가능.
  • MCC 데이터셋 의존성(§5.2) / Reconstruction 선택적·FP 증가(§5.5).

독립 발견 약점

  • ① QoA/평가지표 순환 위험(핵심) — ORTHRUS는 (a) 새 GT 라벨링(41~123 수작업) + (b) precision·MCC를 "옳은 지표"로 규정 + (c) 그 지표에 최적화된 시스템 설계, 세 축 모두 저자 산물. baseline은 저자 새 라벨로 재채점("modifying only the final evaluation step"). QoA 실측(사용자 연구·시간 측정) 전무 — 노드 수·precision으로 대체. GT는 단일 출처 수작업으로 inter-annotator agreement·독립 검증 없음.
  • ② 모든 baseline precision 0.00 — 공정성 red flag — Table 4/5에서 5개 baseline이 거의 모든 데이터셋 precision 0.00, MCC≈0. 경쟁자가 예외 없이 완전 0인 벤치마크는 지표·라벨이 제안기법 쪽으로 구성됐을 가능성을 시사. baseline은 원래 neighborhood/batch로 보정됐는데 strict 라벨로 재채점+재튜닝됐고, MAGIC은 GT 기반 thresholding, SIGL은 재구현본 — 참담한 수치 일부는 프로토콜 산물일 수 있다(추정).
  • ③ "모든 공격 탐지" vs 낮은 recall — 탐지 정의가 관대(노드 1개라도 flag). recall 극히 낮음: E5-CADETS 123 악성 중 TP 2(recall~1.6%), E3-CLEARSCOPE TP 2/41. MCC 0.05~0.16이 6개 중 3개.
  • ④ 재구성 평가 공백 — attack summary graph의 정확도·완결성 정량 평가 없음. FP 시드 강건성 미분석(p가 FP면 추적이 통째로 허구). f_D=Out/In(식9)는 강한 휴리스틱, 최고 f_C 단 하나만 선택 → 다중 진입 공격 놓칠 위험.
  • ⑤ 임계값·클러스터링 취약 — 임계값=validation(benign 하루) 최대 anomaly score. 단일 benign 이상치 섞이면 임계값이 부풀어 recall 붕괴(민감도 분석 없음). K-means k=2 고정은 무공격 윈도우에서도 benign을 강제 2분할 → FP 유발 가능.
  • ⑥ 미미크리/적응형 대응 근거 얇음 — 단일방법(Goyal NDSS'23)·단일 데이터셋(E3-CADETS). Goyal은 이웃 분포 겨냥인데 ORTHRUS는 edge-type reconstruction 기반 → 위협표면이 달라 강건성이 설계가 아닌 우연일 수 있다(추정). 손실 아는 적응형 공격자 미검증.
  • ⑦ 통계적 견고성 부재 — 핵심 수치가 작은 정수 카운트(TP 1~48, FP 0~23), 단일 실행 추정. seed·신뢰구간·분산 없음. Fig.8에서 TP 0↔10 요동은 "튜닝 비의존" 결론과 긴장.

재현·일반화 위협

  • DARPA TC(E3/E5) 단일 의존 — 3 캡처 시스템, 스크립트 benign, 상대적으로 "시끄러운" 공격. 실기업·조용한 지속위협·조직 간 이질성 미검증.
  • train/test가 같은 engagement·같은 호스트(Appendix A) — inductive를 주장하나 cross-host/cross-org 미평가.
  • GT가 재현 병목 — 저자 수작업(Zenodo 공개). baseline 수치는 재튜닝·재구현·GT thresholding 의존 → 독립 재현 시 달라질 여지.

6인용 가치 · 활용 메모

그래서 이 논문을 언제, 어떻게 쓰면 좋을까.

  • 누가/언제:
    • PIDS·프로비넌스 이상탐지 연구자 — 탐지 정확도가 아니라 분석가 검수 부담(QoA) 관점으로 문제를 재정의할 때 인용.
    • 불균형 보안 벤치마크 설계자 — accuracy/F1/AUC 대신 precision·MCC + 검수 노드 수 프레이밍의 근거로.
    • GNN 기반 탐지 실무자 — stateless temporal sampling으로 메모리 절감(~1/3)하는 경량화 레퍼런스.
인용 시 주의 — baseline precision≈0 결과는 엄밀 라벨 재채점의 산물이므로 "기존 시스템이 무용하다"로 오독하지 말 것 — 원 논문들의 자체 F1(0.93~0.99)과는 다른 평가 축이다.
  • 후속 아이디어:
    • QoA를 노드 수 대신 실제 분석가 시간·사용자 연구로 실측.
    • GT에 대한 inter-annotator agreement·독립 재라벨링으로 벤치마크 편향 검증.
    • attack summary graph의 precision/recall 정량 평가와 FP 시드 강건성 실험.
    • 손실을 아는 적응형 공격자 하에서의 강건성, cross-host/cross-org 일반화, 개념 드리프트 재훈련 평가.

정리하면, ORTHRUS의 진짜 기여는 "더 잘 잡는 모델"이 아니라 "무엇을 재야 하는가"라는 질문 자체를 바꾼 것이다. 검수 노드를 5자릿수 줄인 숫자는 강력하지만, 그 숫자가 저자가 세운 지표·라벨 위에서 나왔다는 점은 인용자가 늘 함께 짊어져야 할 각주다.

7. 용어 사전
  • QoA (Quality of Attribution) — 공격 귀속 품질. IDS 출력을 조사·근본원인 파악·오경보 기각하는 분석가 노력으로 정의되는 개념. 형식 공식 없이 검수 노드 수·precision으로 대리 측정.
  • PIDS — Provenance-based Intrusion Detection System.
  • 프로비넌스 그래프 — OS 실행 이력의 방향·속성·동적 그래프(노드=프로세스/파일/netflow, 엣지=시스템콜+타입+타임스탬프).
  • MCC (Matthews Correlation Coefficient) — 불균형에 강건한 상관 기반 지표(식12). 데이터셋 의존적이라 동일 데이터셋 내 비교만.
  • Stateless temporal sampling — 엔티티별 메모리 없이 최근 N개 엣지만 샘플링(식2·3). Kairos 대비 메모리 ~1/3.
  • 평가전략 3종 — Neighborhood(2-hop 조상·후손 전부 악성)/Batch(동시 이벤트 배치 전체)/Source(소스 후손 전부). 셋 다 악성 노드를 수천 개로 과대추정.
  • Attack summary graph — backward/forward 인과 추적의 dependency graph 합집합으로 만든 소수 노드 공격 경로 요약.
  • Automatic Thresholding — validation(benign 하루) 최대 이상점수를 임계값으로 자동 설정.
  • Outlier Clustering — 의심 노드에 K-means(K=2)로 악성/benign anomaly 분리.
  • ORTHRUS-full / -ano — 재구성 포함(full) / 재구성(§4.5) 제외 이상탐지만(ano).
  • "ADP" — 원문 지표명 아님. §5.1 "Attack Detection Performance"(이진 탐지)를 지칭하는 것으로 추정.
